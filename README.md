# Создание разделов

Для автоматизации используются shell-скрипты.
scripts/

mbr_create_max.sh  
создание максимального количества logical разделов в MBR. Использовалось для вычисления максимального количества разделов в MBR

mbr_create_random.sh  
создание logical разделов случайного размера (50–250 MB)

gpt_create_128.sh  
создание GPT разделов

Дополнительные технические заметки находятся в каталоге `docs`.

# Сохранение информации о дисках

Для сохранения информации о структуре дисков использовались стандартные утилиты Linux.

Примеры команд:

```bash
lsblk /dev/sdX
```

```bash
sudo parted /dev/sdX print
```

```bash
sudo fdisk -l /dev/sdX
```

Вывод этих команд сохранён в каталоге:

```
results/
```

## Ограничения таблиц разделов

### MBR

- максимум 4 primary раздела
- либо 3 primary + 1 extended
- внутри extended могут находиться логические разделы

### GPT

- стандартно поддерживает 128 разделов
- размер таблицы можно увеличить, изменив количество partition entries
